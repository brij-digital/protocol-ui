Codama Runtime Demo

How we use Codama as the instruction source of truth

This is the smallest concrete walkthrough of the boundary we ended up with. Codama carries instruction execution truth. Our runtime keeps only deterministic compute and write preparation on top of that.

Open live runner flow Open raw operations

Live flow

  1. Resolve pair inputs like USDC -> SOL from indexed token and pool data.
  2. Run a deterministic quote compute.
  3. Draft the Orca swap from Codama-backed instruction metadata.
  4. Optionally submit in wallet from the Runner view.

Recommended demo in the app: Draft Swap on Orca.

Boundary

Codama Instruction-level source of truth accounts, signers, PDA/default accounts
Runtime spec Deterministic compute and write prep quote math, draft construction

Two files to inspect

Codama IDL /idl/orca_whirlpool.codama.json

Instruction truth: accounts, defaults, PDA metadata.

 Runtime Spec /idl/orca_whirlpool.runtime.json

Compute and contract-write capabilities only.
The remaining custom layer on our side is mostly quote/preview logic and transaction-envelope logic. That is why Codama instruction plans feel like the natural next place to reduce our runtime surface even further.
