How we use Codama as the instruction source of truth

This page shows one concrete Orca swap flow. Codama carries the instruction-level truth. Our runtime only adds deterministic quote computation and swap draft preparation.

Live flow

  1. Start from a fixed pair and fixed mints: USDC -> SOL.
  2. Run a deterministic quote compute for 1 USDC.
  3. Draft the Orca swap from Codama-backed instruction metadata.

The pair and the mints are intentionally fixed here so the demo stays focused on Codama + runtime, not on discovery.

How the runtime spec works

        

          After pushing instruction-level execution into Codama, our runtime spec stays very small.
          It does not redefine the Orca instruction. It only declares the deterministic work needed
          around that instruction.
        

        

          

            views
            quote_exact_in
            
Runs deterministic quote logic and produces values such as estimated_out and minimum_out.

          

          

            writes
            swap_exact_in
            
Uses Codama’s instruction metadata plus the derived values to prepare the final swap draft.

          

          

            load
            decode_account, ata, pda
            
Loads only the extra runtime context that is still needed and not already implied by Codama.

          

          

            transforms
            shared deterministic fragments
            
Defines reusable transform blocks that a read or write can reference instead of repeating the same logic inline.

          

        

      


      

        For this Orca flow, Codama already covers the instruction shape and account model. The small
        layer left on our side is deterministic quote logic plus draft preparation around that
        instruction.
