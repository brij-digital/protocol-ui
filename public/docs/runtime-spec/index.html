Runtime Spec V1 Reference

Program-specific runtime logic in AppPack

This page documents the current solana-agent-runtime.v1 contract and where the program-specific logic lives on the runtime side.

Scope boundary

Codama
  • instruction shape
  • named accounts
  • signers
  • fixed/default accounts
  • PDA-backed defaults when declared
Runtime spec
  • deterministic read operations
  • write preparation
  • reusable transform fragments
  • load steps outside Codama
  • small transaction-envelope logic

Shared operation core

AttributeRequiredTypeMeaning
loadNoarrayArray of loadStepSpec.
transformNoarrayArray of string references to top-level transforms.

Both readSpec and writeSpec share this preparation phase. They diverge only at the end: reads return a typed result, while writes materialize args and accounts.

readSpec

AttributeRequiredTypeMeaning
inputsNoobjectMap of input name to type string.
outputNoobjectTyped output contract.

writeSpec

AttributeRequiredTypeMeaning

loadNoarrayArray of loadStepSpec.

remaining_accountsNostring or arrayString reference or explicit meta list.
preNoarrayArray of preInstructionSpec.
postNoarrayArray of postInstructionSpec.

Write inputs are not declared in the runtime file. They come from the referenced Codama instruction, filtered down to the Codama args/accounts still referenced through $input.* inside the write.

transformSpec

Top-level transforms is the only place where transform steps are declared. Each entry is an array of transformStepSpec referenced from reads or writes by name.

Concrete Orca example

In the Orca pack, quote_exact_in decodes the Whirlpool account, derives direction, tick arrays, estimated_out, and minimum_out. Then swap_exact_in uses Codama instruction metadata plus these derived values to prepare the final write. 

{
  "instruction": "swap_v2",
  "load": [
    {
      "name": "wallet",
      "kind": "wallet_pubkey"
    },
    {
      "name": "whirlpool_data",
      "kind": "decode_account",
      "address": "$input.whirlpool",
      "account_type": "Whirlpool"
    }
  ],
  "transform": [
    {
      "name": "a_to_b",
      "kind": "compare.equals",
      "left": "$whirlpool_data.token_mint_a",
      "right": "$input.token_in_mint"
    },
    {
      "name": "minimum_out",
      "kind": "coalesce",
      "values": ["$other_amount_threshold"]
    }
  ],
  "args": {
    "amount": "$input.amount_in",
    "other_amount_threshold": "$other_amount_threshold"
  }
}
