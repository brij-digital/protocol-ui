AppPack Runtime Spec

Where the program-specific runtime logic lives

AppPack keeps instruction-level truth in Codama, indexed reads in the indexing spec, and only a narrow deterministic execution layer in the runtime spec.

The split

Codama
instruction source of truth

Instruction shape, named accounts, signer metadata, fixed defaults, PDA-backed defaults.

Indexing spec
reads and discovery

Search, feeds, ranking, series, and all other indexed read semantics.

Runtime spec
deterministic execution layer

Deterministic compute, write preparation, and the small transaction envelope around a write.

What the program maintainer writes

Program-specific runtime logic lives in a per-protocol file such as public/idl/orca_whirlpool.runtime.json. That file is provided by the pack maintainer.

computes

Named deterministic protocol computations such as quotes, thresholds, and typed previews.

contract_writes

Instruction preparation using Codama metadata plus runtime inputs and computed values.

resolve

Only the extra runtime context still needed outside Codama, such as account decode or ATA derivation.

pre / post

The remaining transaction-envelope logic, such as ATA creation, WSOL sync, and cleanup.

Minimal file shape

{
  "$schema": "/idl/solana_agent_runtime.schema.v1.json",
  "schema": "solana-agent-runtime.v1",
  "computes": {
    "quote_exact_in": { "...": "..." }
  },
  "contract_writes": {
    "swap_exact_in": { "...": "..." }
  }
}

The runtime spec deliberately does not restate the raw instruction schema. Codama already owns that.

Concrete Orca example

quote_exact_in

Decodes the Whirlpool account, computes swap direction, tick arrays, estimated_out, and minimum_out.

swap_exact_in

Uses Codama’s swap_v2 metadata plus computed values to prepare the final draft.

{
  "resolve": [
    {
      "name": "wallet",
      "kind": "wallet_pubkey"
    },
    {
      "name": "whirlpool_data",
      "kind": "decode_account",
      "address": "$input.whirlpool",
      "account_type": "Whirlpool"
    }
  ],
  "compute": [
    {
      "name": "a_to_b",
      "kind": "compare.equals",
      "left": "$whirlpool_data.token_mint_a",
      "right": "$input.token_in_mint"
    },
    {
      "name": "minimum_out",
      "kind": "coalesce",
      "values": ["$other_amount_threshold"]
    }
  ]
}

What stays out of the runtime spec

Keep in Codama

Instruction structure, account metadata, signer metadata, fixed/default accounts, PDA-backed defaults.

Keep in the indexing spec

Discovery, search, feeds, ranking, and time-series semantics.

Keep out of both

Any flow that requires “transaction A, then read live state, then transaction B” belongs in a higher-level runtime.
