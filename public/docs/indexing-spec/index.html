Declarative Decoder Runtime V1

Ingest specs and indexed reads specs

declarative-decoder-runtime.v1 now covers two related document shapes: *.ingest.json for ingest and canonical emission, and *.indexed-reads.json for indexed querying and projection.

Architecture

Chain events -> [Ingest spec] -> Raw canonical facts -> Postgres
                                                     |
Client request -> [Indexed reads spec] -> Filter/Sort/Aggregate -> Response

Design rules

Ingest pipelines store only raw canonical facts
  • Direct values from event and account payloads
  • Small pure normalization like unit conversion or renaming
  • No joins such as state.lookup
  • No external calls such as rpc.token_supply
  • No baked business metrics like price, liquidity, or market cap
Indexed reads expose stored facts
  • Filter, sort, aggregate, reshape
  • No enrichment at query time
Runtime views own enrichment
  • Price
  • Market cap
  • Liquidity
  • Any richer derived metric computed at read time

Registry fields

{
  "id": "pump-amm-mainnet",
  "ingestSpecPath": "/idl/pump_amm.ingest.json",
  "indexedReadsPath": "/idl/pump_amm.indexed-reads.json"
}
  • ingestSpecPath is used by the Carbon plan compiler.
  • indexedReadsPath is used by the canonical view runner and query layer.
  • indexingSpecPath is deprecated and kept only as fallback.

Ingest Spec

The ingest document owns decoding, routing, canonical extraction, normalization, and emission.

Top-level shape

{
  "schema": "declarative-decoder-runtime.v1",
  "protocolId": "pump-amm-mainnet",
  "programId": "$protocol.programId",
  "decoderArtifacts": {},
  "sources": {},
  "matchRules": [],
  "pipelines": {}
}

decoderArtifacts

{
  "pump_swap": {
    "kind": "generated_idl_decoder",
    "family": "codama",
    "codamaPath": "/idl/pump_amm.codama.json",
    "artifact": "pump_swap"
  }
}

sources

{
  "pump_swap_crawler": {
    "kind": "rpc_transaction_crawler",
    "programId": "$protocol.programId",
    "commitment": "confirmed",
    "decoderRef": "pump_swap",
    "decodeTargets": ["instruction", "log_event"]
  }
}

matchRules

[
  {
    "source": "pump_swap_crawler",
    "match": { "decodedType": "event", "name": "BuyEvent" },
    "pipeline": "trade_buy"
  }
]

pipelines

Each pipeline has extract -> compute -> emit. In new ingest specs, no resolve steps.

{
  "trade_buy": {
    "extract": {
      "pool": "$decoded.pool",
      "user": "$decoded.user",
      "side": "buy",
      "baseAmountAtomic": "$decoded.base_amount_out",
      "quoteAmountAtomic": "$decoded.quote_amount_in"
    },
    "compute": [
      {
        "name": "baseAmountUi",
        "compute": "math.amount_ui",
        "amount": "$ctx.baseAmountAtomic",
        "decimals": 9
      }
    ],
    "emit": {
      "record": {
        "recordName": "BuyEvent",
        "subjectId": "$ctx.pool",
        "payload": { "...": "raw facts only" }
      }
    }
  }
}

Allowed compute ops in ingest

  • math.amount_ui
  • math.add, math.sub, math.mul, math.safe_div
  • compare.gt, compare.equals
  • logic.if
  • coalesce

Indexed Reads Spec

The indexed-reads document owns queryable operations over canonical records and state.

Top-level shape

{
  "schema": "declarative-decoder-runtime.v1",
  "protocolId": "pump-amm-mainnet",
  "programId": "$protocol.programId",
  "operations": {}
}

operations

{
  "trade_feed": {
    "index_view": {
      "kind": "search",
      "canonical": {
        "source": "indexed_idl_records",
        "protocol_id": "pump-amm-mainnet",
        "record_kind": "event",
        "record_name": ["BuyEvent", "SellEvent"]
      },
      "projection": {
        "projection_kind": "feed",
        "sort_by": [{ "field": "eventTime", "dir": "desc" }]
      }
    }
  }
}

Projection kinds

KindDescriptionAllowed ops
feedChronological event streamsort, limit
snapshotLatest state per subjectfilter, sort
seriesTime-bucketed aggregationaggregate, bucket
rankingScored or sorted itemsfilter, sort, score
searchFilterable state queriesfilter, sort

Deprecated features

  • resolve steps in ingest pipelines are deprecated.
  • indexingSpecPath in registry is deprecated.
  • Top-level projectionSpecs in ingest is deprecated.
